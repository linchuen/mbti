# formation-process-guide.md

## 頁面目的

此頁面補上「如何判斷人格形成流程」的教學。

重點：

* 教使用者觀察自己的認知處理順序
* 以流程判斷，不以人格特質詞判斷
* 產出「候選認知功能路徑」，不直接下類型結論

不得做人格測驗結果頁、不得直接判定使用者是某一類型。

## 2. 判斷前提（先看認知，不看標籤）

### 目的

建立判斷原則，避免使用特質形容詞。

### UI

三條原則卡片（條列）

### 固定原則

* 以「處理資訊方式」為判斷對象
* 以「長期重複模式」為判斷證據
* 以「壓力下的反應偏移」做交叉驗證

---

## 3. 人格形成流程判斷（四步）

### UI

步驟流程（stepper 或 ordered list）

### 固定主流程文字（不可修改）

感知方式 → 判斷方式 → 功能排序 → 人格傾向

### 每步內容要求

#### Step 1 感知方式

判斷使用者習慣先抓取：

* 外部即時訊號（Se / Ne 傾向）
* 內部既有脈絡（Si / Ni 傾向）

#### Step 2 判斷方式

判斷使用者決策時優先：

* 邏輯一致性（Ti / Te 傾向）
* 價值一致性（Fi / Fe 傾向）

#### Step 3 功能排序

依「自然使用頻率 + 壓力時失衡點」推估：

* 優勢功能（dominant）
* 輔助功能（auxiliary）
* 第三功能（tertiary）
* 劣勢功能（inferior）

#### Step 4 人格傾向

輸出為「認知運作傾向摘要」，不得輸出最終類型判定。

---

## 4. 判斷紀錄表（可重複使用）

### UI

表格或卡片清單，至少 3 筆觀察紀錄

### 每筆欄位

* 情境描述（做了什麼決策）
* 先感知到什麼
* 最後用什麼標準下判斷
* 事後壓力反應

### 說明

用多次紀錄找穩定模式，不得用單一事件下結論。

---

## 5. 常見誤判修正

### UI

左右對照列表

左：常見誤判  
右：正確修正方式

### 至少包含以下三項

* 「我外向，所以一定是 Fe/Se」→ 需回到判斷標準，不看社交表現
* 「我重感受，所以一定是 Fi」→ 需區分個人價值判斷與情緒強度
* 「我工作理性，所以一定是 T 主導」→ 需分辨角色要求與自然偏好
